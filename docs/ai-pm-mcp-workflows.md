# AI PM 情报雷达：MCP 分析工作流模板（5-8条）

> 目标：把“查什么 / 怎么查 / 怎么复盘”固化成可复制的对话工作流，减少随机发挥，提升稳定产出。
>
> 使用前提：
> - 你有足够的历史数据（至少 7 天）才能做周环比/趋势。
> - 如果你的爬虫跑在 GitHub Actions 且数据在远程存储，本地 MCP 侧建议先执行一次 `sync_from_remote`（最近 7-30 天）。

---

## Workflow 1：跨平台聚合去重（看“同一事件”覆盖面）

**适用**：快速识别“一个事件被多个平台同时报道”的强信号，减少重复阅读。

**工具链**：
- `aggregate_news`

**对话模板**（复制即用）：
1. “把**今天**的新闻做一次**跨平台聚合去重**，相似度阈值 0.7，返回前 50 条，给我一个按综合权重排序的列表。”
2. “从聚合结果里挑出**跨平台覆盖 ≥3** 的事件，分别给：一句话摘要 + 影响判断（产品/竞争/供应链/法规）+ 我该跟进谁（公司/机构/媒体/账号）。”

**可调参数建议**：
- `similarity_threshold`: 0.65（更宽松）/ 0.75（更严格）
- `limit`: 30（更聚焦）/ 100（更全）

---

## Workflow 2：周环比（热点迁移 / 新增主题 / 消失主题）

**适用**：每周一次“宏观扫描”——本周相较上周发生了什么变化。

**工具链**：
- `compare_periods`（compare_type=overview / topic_shift）

**对话模板**：
1. “对比 `last_week` vs `this_week`，做一次 `compare_periods`：先用 `overview`，top_n=15。”
2. “再对比一次，用 `topic_shift`，top_n=20。把**新出现**、**上升**、**下降**主题各列 10 个，并解释可能原因（政策/产品发布/资本/舆情/技术突破）。”
3. “对每个‘新出现主题’，帮我生成一个**跟进清单**：要查的 3 个问题 + 可能的信号源（媒体/公司/监管/社区）。”

---

## Workflow 3：平台温差（“同一话题”各平台关注差异）

**适用**：同一话题，在“国内热榜 / 海外媒体 / 社区”出现强温差时，往往意味着信息不对称或叙事转向。

**工具链**：
- `analyze_data_insights`（insight_type=platform_compare）
- 可选：`get_platform_activity_stats`

**对话模板**：
1. “对比各平台对话题 **<topic>** 的关注度（platform_compare），时间范围最近 7 天。”
2. “如果某个平台显著更高，给我：可能原因（用户结构/媒体属性/监管差异/产品阶段）+ 我下一步应该在哪里补信息（建议 3 个平台/社区）。"

**提示**：
- `<topic>` 可以是：`AI glasses`、`Ray-Ban Meta`、`Oura`、`XR`、`on-device`、`agent` 等。

---

## Workflow 4：弱信号雷达（爆火检测 + 潜力预测 + 证据回捞）

**适用**：捕捉突然爆发的主题，并快速回捞证据标题，判断是否值得立项/跟进。

**工具链**：
- `analyze_topic_trend`（analysis_type=viral / predict）
- `search_news` / `search_rss`

**对话模板**：
1. “检测今天有哪些突然爆火的话题（viral），threshold=3.0，time_window=24。”
2. “对 TOP 5 爆火词分别：回捞相关标题（search_news，包含 URL），并从 RSS 也搜索一遍（search_rss，最近 7 天）。”
3. “按‘产品落地可能性/竞争影响/供应链影响/合规风险’四维打分（1-5），给出是否进入重点追踪列表的结论。”

---

## Workflow 5：竞品追踪（AI 眼镜 / 耳机 / 戒指：事件-时间线）

**适用**：把“竞品动态”从碎片标题变成可执行的时间线（发布、量产、渠道、定价、体验）。

**工具链**：
- `search_news`（热榜）
- `search_rss`（媒体/RSS）
- 可选：`find_related_news`（相似新闻扩展）

**对话模板**：
1. “搜索最近 14 天关于 **<brand_or_product>** 的新闻（search_news），需要链接，返回前 50 条。”
2. “在 RSS 里也搜索最近 30 天（search_rss），需要摘要，返回前 50 条。”
3. “把结果整理成：时间线（按日期）+ 关键事实（规格/价格/渠道/供应链/合规）+ 风险点 + 我们可以抄的 3 个体验点。”

**建议追踪对象示例**：
- AI 眼镜：Meta Ray-Ban、XREAL、Rokid、RayNeo、Google/Android XR、Apple（Vision/眼镜线索）
- 耳机：AirPods、FreeBuds、实时翻译、端侧会议纪要
- 戒指/腕上：Oura、Galaxy Ring、WHOOP、Garmin、Apple Watch（健康指标与订阅策略）

---

## Workflow 6：供应链 / 合规 / 量产信号（“能不能卖”与“何时卖”）

**适用**：从标题中提取“量产与合规”信号，支持产品执行判断（成本/交付/渠道/法规）。

**工具链**：
- `search_news` / `search_rss`
- `compare_periods`（可选：看信号密度变化）

**对话模板**：
1. “搜索最近 30 天包含关键词：FCC / CE / 量产 / 试产 / 出货 / 备案 / 认证 / BOM / 供应链 的新闻（热榜+RSS都要），需要链接。”
2. “把每条信号归类为：合规 / 量产 / 渠道 / 定价，并标注‘可信度’（高/中/低）与‘下一步验证动作’（要找谁/看什么文件/等什么公告）。”

---

## Workflow 7：前沿能力 & 范式跃迁（模型能力 → 产品机会）

**适用**：把“模型能力变化”翻译成“可穿戴/端侧/交互产品”的机会与约束。

**工具链**：
- `search_rss`（建议包含 arXiv / HN / 媒体）
- `analyze_topic_trend`（trend / lifecycle）

**对话模板**：
1. “在 RSS 中搜索最近 14 天游走在前沿的能力关键词：multimodal / on-device / small model / streaming / realtime / tool calling / agent（需要摘要）。”
2. “对其中 3 个最重要的能力点：分别给出 2 个可穿戴场景落地构想（交互/功耗/隐私/延迟约束）+ 1 个落地阻碍 + 你建议我验证的最小实验。”

---

## Workflow 8：每周深度（自动摘要 + 追踪清单）

**适用**：形成“周报底稿”，再在此基础上做人工补充与决策。

**工具链**：
- `generate_summary_report`（weekly）
- 可选：`get_rss_feeds_status`

**对话模板**：
1. “生成本周热点总结（weekly）。”
2. “基于周报内容，给我一份‘下周追踪清单’：Top 10 主题，每个主题 1 句结论 + 1 个待验证问题 + 2 个建议信息源。”
